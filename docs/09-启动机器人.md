# 第九步：启动机器人

> 一切就绪，终于可以启动机器人了！
>
> 📍 **在哪操作：🟢 树莓派窗口**（提示符 `pi@pi-zero:~ $`）

## 9.1 测试意图解析

> 先确认机器人能听懂你的指令。这一步不涉及语音，直接文字测试。

在树莓派窗口输入以下命令（复制整段）：

```bash
cd ~/robot && python3 -c "
from intent_parser import IntentParser
p = IntentParser()
tests = ['播放大海', '下一首', '暂停', '今天心情怎么样']
for t in tests:
    r = p.parse(t)
    print(f'{t} → {r[\"intent\"]}')
"
```

> 注意：这一步需要网络，DeepSeek 在云端分析你的指令。

**✅ 看到类似这样的输出：**

```
播放大海 → play_music
下一首 → next_song
暂停 → pause
今天心情怎么样 → chat
```

说明 DeepSeek Key 配置正确，机器人能听懂指令。

> 先在终端里直接运行，看看有没有报错。

在树莓派窗口输入：

```bash

**✅ 看到类似这样：**

```

  意图 ✅

> 这时机器人已经在运行了。注意这个终端被占住了，不要关。

## 9.3 喊出第一声

> 准备喊出你的第一声"土豆"！

**确保：**
- ✅ 音箱通电且音量适中
- ✅ 麦克风插着
- ✅ 你离麦克风不远（半米以内）

**对着麦克风说：**

> **「土豆放一首大海」**

### 机器人会经历以下流程：

```
① 你说"土豆放一首大海"
② 机器人检测到声音 → 开始录音
③ 百度语音识别 → 得到文字"土豆放一首大海"
④ 分离唤醒词 → 提取出指令"放一首大海"
⑤ DeepSeek 解析意图 → {"intent":"play_music", "params":{"keyword":"大海"}}
⑥ 执行器搜索"大海" → 找到歌曲
⑦ 音箱开始播放音乐 🎵
```

**✅ 听到音乐 → 成功了！🎉**

### 测试完按 `Ctrl+C` 停止机器人

> 按住 `Ctrl` 键不放，然后按 `C` 键。机器人会退出，回到提示符。

## 9.5 配置开机自启

> 让树莓派插电后自动运行机器人，不用手动启动。

**第一步：创建自启配置文件**

在树莓派窗口输入（复制整段命令，粘贴进去）：

```bash
cat > ~/.config/systemd/user/robot.service << 'EOF'
[Unit]
Description=Robot Service
After=network.target sound.target netease-api.service

[Service]
Type=simple
ExecStart=/usr/bin/python3 %h/robot/main.py
WorkingDirectory=/home/pi/robot
Restart=on-failure
RestartSec=10

[Install]
WantedBy=default.target
EOF
```

按回车执行（没有输出是正常的）。

**第二步：启用自启**

先执行：

```bash
systemctl --user daemon-reload
```

再执行：

```bash
systemctl --user enable robot.service
```

最后启动服务：

```bash
systemctl --user start robot.service
```

**第三步：验证是否启动成功**

```bash
systemctl --user status robot.service
```

**✅ 看到 `active (running)`（绿色字）→ 开机自启配置完成！**

> 如果看到 `inactive (dead)` 或 `failed`（红色字），说明启动失败了。检查一下：
> - 有没有执行过 9.5 第一步（创建 service 文件）？
> - 可以看日志排查：`journalctl --user -u robot.service -n 20`

## 9.6 重启验证

> 拔电重启，看看机器人会不会自动启动。

```bash
sudo reboot
```

树莓派会重启，SSH 连接会断开。**等 30 秒左右**让它启动完成。

**重新 SSH 连上树莓派（按第4步的方法）：**

```bash
ssh pi@192.168.1.100
```

密码：`raspberry`

**连上后检查机器人状态：**

```bash
systemctl --user status robot.service
```

**✅ 看到 `active (running)` → 🎉 🎉 🎉 全部完成！**

从拆快递到完整的 AI 语音机器人，你已经走完了全程。
