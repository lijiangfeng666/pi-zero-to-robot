## 🔧 常用维护命令

> 在树莓派窗口（SSH 连接后）执行。

### 查看机器人状态

```bash
systemctl --user status robot.service
```

看到 **`active (running)`**（绿色）说明机器人在正常运行。

### 查看运行日志

```bash
# 查看最近 30 行日志
journalctl --user -u robot.service -n 30

# 实时看日志（按 Ctrl+C 退出）
journalctl --user -u robot.service -f

# 看今天所有的日志
journalctl --user -u robot.service --since today
```

### 管理机器人

```bash
# 重启机器人（改完配置后需要执行）
systemctl --user restart robot.service

# 停止机器人
systemctl --user stop robot.service

# 禁用开机自启
systemctl --user disable robot.service

# 启用开机自启
systemctl --user enable robot.service
```

### 管理音乐服务

```bash
# 查看音乐服务状态
systemctl --user status netease-api.service

# 重启音乐服务
systemctl --user restart netease-api.service
```

## ⚙️ 常用参数调优

在 `~/robot/.env` 文件中修改参数，改完**重启机器人**生效。

### 如何修改 .env 文件

```bash
# 连上树莓派后
nano ~/robot/.env
```

改完按：`Ctrl+X` → `Y` → `回车`（跟第6步一样）

### 参数说明

| 参数 | 默认值 | 作用 | 什么时候调 |
|------|--------|------|-----------|
| `WAKE_WORD` | 土豆 | 唤醒词 | 想改成"你好""小爱"等 |
| `VAD_THRESHOLD` | 2500 | 麦克风灵敏度，**越小越灵敏** | **说"土豆"没反应时**改小到 300-800 |
| `MIC_GAIN` | 3000 | 麦克风增益，**越大越灵敏** | 麦克风声音太小，改到 5000-8000 |
| `VOLUME` | 1.0 | 默认音量（0.0~1.0） | 觉得默认太大/太小 |
| `WAKE_RECORD_SECONDS` | 1.5 | 唤醒后录音时长 | 说话慢的人可以改到 2.5 |
| `RECORD_SECONDS` | 2.0 | 普通指令录音时长 | 指令比较长可以改到 3.0 |
| `COMMAND_TIMEOUT` | 4 | 超时秒数 | 说完唤醒词后等待指令的最大时间 |

**常见调优场景：**

> 🔹 机器人总是**听不到我说话**：把 `VAD_THRESHOLD` 从 2500 降到 **500**
> 🔹 机器人**太灵敏**，没人说话也在录：把 `VAD_THRESHOLD` 从 2500 升到 **5000**
> 🔹 机器人**总觉得我叫它**（误唤醒）：把 `VAD_THRESHOLD` 调高到 8000
> 🔹 我**说话比较慢**：把 `WAKE_RECORD_SECONDS` 改为 2.5，`RECORD_SECONDS` 改为 3.0

**✅ 改完 `.env` → 运行下面命令生效：**

```bash
systemctl --user restart robot.service
```

## 📤 更新代码

当 GitHub 上的代码有更新时：

### 方法一：重新下载全部代码（推荐，最稳）

```bash
# 在电脑窗口操作（不是树莓派窗口！）
# 先看一下你的代码在电脑的什么位置
# 一般在第6步克隆到了桌面

cd ~/Desktop/pi-zero-to-robot    # 改你的实际路径
git pull                          # 拉取最新代码

# 传到树莓派（替换 IP 地址）
scp main.py config.py voice.py music_player.py netease.py executor.py intent_parser.py pi@192.168.1.100:~/robot/

# 重启机器人
ssh pi@192.168.1.100 "systemctl --user restart robot.service"
```

### 方法二：只更新单个文件

```bash
# 比如只改进了 main.py，只传这一个文件
scp main.py pi@192.168.1.100:~/robot/
ssh pi@192.168.1.100 "systemctl --user restart robot.service"
```

## 🔌 树莓派日常注意事项

| 注意 | 说明 |
|------|------|
| **不要直接拔电源** | 关机器前先 `sudo shutdown -h now`，等绿灯不闪了再拔 |
| **SD 卡**是唯一硬盘 | SD 卡寿命有限，**尽量别频繁写文件** |
| **保持散热** | 4B 发热较大，建议加散热片或小风扇 |
| **定期重启** | 长时间运行后，`sudo reboot` 清一下内存 |
| **不要同时插太多 USB** | 树莓派供电有限，插太多设备可能供电不足 |
