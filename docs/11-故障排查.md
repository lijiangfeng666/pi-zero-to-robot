# 第十一步：故障排查

### 说"土豆"没反应

```
① 靠近麦克风大声一点（半米以内）
② 在树莓派窗口输入下面命令看日志：
   journalctl --user -u robot.service -n 20
   按回车 —— 如果显示 inactive 说明机器人没在跑
③ 测试麦克风：
   在树莓派窗口输入：arecord -d 3 /tmp/t.wav
   按回车 → 对着麦克风说话 → 再输入：aplay /tmp/t.wav
   按回车 —— 听不到回放说明麦克风有问题
④ 调低灵敏度：
   在树莓派窗口输入：nano ~/robot/.env
   找到 VAD_THRESHOLD=2500，改成 300
   按 Ctrl+X → Y → 回车保存
   再输入：systemctl --user restart robot.service
```

### 播放音乐没声音

```
① 检查音乐服务：
   在树莓派窗口输入：curl http://127.0.0.1:3000
   按回车 —— 看到 {} 数据说明服务正常
② 检查音箱：
   在树莓派窗口输入：aplay -l
   按回车 —— 看到 card 说明音箱被识别
③ 调音量：
   在树莓派窗口输入：pactl set-sink-volume 0 70%
   按回车
```

### 语音识别总是返回 None

```
① 检查网络：
   在树莓派窗口输入：ping baidu.com
   按回车 —— 有返回数据说明网络通
② 检查 Key：
   在树莓派窗口输入：nano ~/robot/.env
   确认 BAIDU_APP_ID、BAIDU_API_KEY、BAIDU_SECRET_KEY 都填了
③ 靠近麦克风再试一次
```

### 开机不自启

```
① 检查服务状态：
   在树莓派窗口输入：systemctl --user status robot.service
   按回车
② 查看启动日志：
   在树莓派窗口输入：journalctl --user -u robot.service -n 20 --no-pager
   按回车
```

### 快速自检命令

**✅ 复制下面整段命令，粘贴到树莓派窗口，按回车——一键检查机器人状态：**

```bash
echo "=== 文件 ===" && ls ~/robot/*.py 2>&1 | wc -l && echo "个.py" && echo "=== 麦克风 ===" && arecord -l 2>&1 | head -2 && echo "=== 内存 ===" && free -h | grep Mem && echo "=== 服务 ===" && systemctl --user is-active robot.service 2>/dev/null || echo "未配置"
```
