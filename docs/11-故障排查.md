# 第十一步：故障排查

> 遇到问题先看这里。按**症状**找对应的解决方法。

---

## 📖 怎么看日志

> 大部分问题都可以从日志中找到原因。

```bash
# 看机器人最近 30 行日志
journalctl --user -u robot.service -n 30 --no-pager

# 实时看日志（按 Ctrl+C 退出）
journalctl --user -u robot.service -f
```

## 🚨 按症状排查

### 说"土豆"没反应 / 机器人不理我

```
最可能的原因：麦克风没识别到你的声音
```

**① 靠近麦克风，大声说"土豆"**
机器人对音量有要求，正常说话距离在 **半米以内**。

**② 看日志确认机器人是否在运行**
```bash
systemctl --user status robot.service
```
如果显示 `inactive (dead)` → 机器人没在跑，执行 `systemctl --user start robot.service`

**③ 测试麦克风是否正常**
```bash
arecord -d 3 /tmp/t.wav && aplay /tmp/t.wav
```
录 3 秒 → 回放 → 听不到声音说明麦克风有问题：
- 换一个 USB 口插
- 确认是 **USB 麦克风**，不是摄像头自带的麦克风
- 在树莓派上重启音频服务：`sudo systemctl restart pipewire`

**④ 调低 VAD_THRESHOLD（麦克风灵敏度）**
```bash
nano ~/robot/.env
```
把 `VAD_THRESHOLD=2500` 改成 **`VAD_THRESHOLD=300`**，保存后：
```bash
systemctl --user restart robot.service
```

> `VAD_THRESHOLD` 越小越灵敏。2500 适合安静环境，环境噪音大或麦克风差就调低。

**⑤ 调高 MIC_GAIN（麦克风增益）**
在 `.env` 把 `MIC_GAIN=3000` 改成 **`MIC_GAIN=8000`**，重启机器人。

### 机器人能听到我说话，但回答"没识别到"

```
原因：语音识别（STT）返回空，通常是网络或 Key 问题
```

**① 检查网络**
```bash
ping baidu.com
```
如果没通 → 树莓派没连网，检查 WiFi 配置。

**② 检查百度 Key**

```bash
cd ~/robot && python3 -c "
from dotenv import load_dotenv
import os
load_dotenv()
print('AppID:', os.getenv('BAIDU_APP_ID'))
print('API Key:', os.getenv('BAIDU_API_KEY'))
print('Secret Key:', os.getenv('BAIDU_SECRET_KEY'))
"
```

如果看到任何一个是 `None` 或空的 → `.env` 文件里 Key 没填对。

```bash
nano ~/robot/.env     # 检查并修正
```

**③ 验证 Key 是否有效**

```bash
cd ~/robot && python3 -c "
from aip import AipSpeech
from dotenv import load_dotenv
import os
load_dotenv()
c = AipSpeech(os.getenv('BAIDU_APP_ID'), os.getenv('BAIDU_API_KEY'), os.getenv('BAIDU_SECRET_KEY'))
r = c.synthesis('测试', 'zh', 1, {'vol': 5})
print('Key 有效' if not isinstance(r, dict) else 'Key 无效，请检查')
"
```

看到 `Key 无效` → 去百度智能云控制台检查你的 Key 是否正确。

### 机器人能识别文字，但不执行指令

```
原因：DeepSeek API Key 有问题
```

**① 检查 DeepSeek Key**

```bash
cd ~/robot && python3 -c "
from dotenv import load_dotenv
import os
load_dotenv()
key = os.getenv('DEEPSEEK_KEY')
print('Key 存在' if key and key != '你的DEEPSEEK_API_KEY' else 'Key 没配置')
"
```

**② 如果 Key 存在但不工作 → 去 DeepSeek 平台检查余额**

```
https://platform.deepseek.com/
→ 左边点「Usage」或「用量」
→ 看看还有没有额度
```

### 音乐播放没声音

```
原因：音箱没识别、音量太小、或音乐服务没启动
```

**① 确认音箱被识别**
```bash
aplay -l
```
如果显示 `no soundcards found` → 音箱没插好，拔插一下 USB 口。

**② 确认音乐服务在运行**
```bash
systemctl --user status netease-api.service
```
显示 `active (running)` 即正常。如果 `failed` → 重启：
```bash
systemctl --user restart netease-api.service
```

**③ 调音量**
```bash
# 调到 70%
pactl set-sink-volume 0 70%
```

**④ 测试音箱是否出声**
```bash
arecord -d 3 /tmp/tt.wav   # 录自己说话
aplay /tmp/tt.wav           # 回放
```
听不到 → 音箱坏了或没通电。

### 开机不自启

> 每次开机后都要手动 `python3 main.py` 才能运行？

```bash
# 检查服务文件是否存在
ls -la ~/.config/systemd/user/robot.service

# 检查是否启用
systemctl --user is-enabled robot.service

# 启用开机自启
systemctl --user enable robot.service
systemctl --user daemon-reload
```

**如果还是不自启，检查开机启动日志：**
```bash
journalctl --user -u robot.service -n 20 --no-pager
journalctl --user -u robot.service --since today
```

### SSH 连不上树莓派

| 错误 | 原因 | 解法 |
|------|------|------|
| `Connection timed out` | 树莓派没开机或 IP 不对 | 检查电源灯是否亮，重新找 IP |
| `Connection refused` | SSH 服务没开 | SD 卡插电脑重新烧录（记得勾 SSH） |
| `Permission denied, please try again` | 密码错误 | 密码是 `raspberry`（全小写） |
| `Host key verification failed` | 重装过系统 | 在电脑窗口执行：`ssh-keygen -R 你的树莓派IP` |

### 终端相关小白问题

| 问题 | 原因 | 解法 |
|------|------|------|
| 复制了命令但**贴不进去** | 终端不支持 Ctrl+V | **右键**点终端空白处 |
| 输入了密码但**屏幕不显示** | 正常！Linux 不显示密码字符 | 正常输入后按回车 |
| 命令跑了一半卡住了 | 可能在下载，需要等待 | 等 1 分钟，还是不动按 `Ctrl+C` 取消重试 |
| 出现 `[Y/n]` 提示 | 在问你是否继续 | 按 `Y` 再按回车 |
| 出现 `[sudo] password for pi:` | 需要 sudo 密码 | 输入 `raspberry`（不显示是正常的） |
| 不小心**关掉了树莓派窗口** | SSH 断开了 | 重新 SSH 连一次（第4步） |
| **看不懂错误信息**（一堆英文） | 正常的 | 复制红色的关键词去搜索，或者截图问 AI |
| `Permission denied` | 没有权限 | 命令前面加 `sudo` 再试 |
| `command not found: xxx` | 这个软件没装 | 跳过此步骤，或者用 `sudo apt install xxx` 装 |

## ✅ 快速自检命令

一键检查机器人整体状态，**复制整段粘贴到树莓派窗口**：

```bash
echo "========== 文件检查 ==========" && \
ls ~/robot/*.py 2>&1 | wc -l && \
echo "个 .py 文件" && \
echo "" && \
echo "========== 麦克风 ==========" && \
arecord -l 2>&1 | head -3 && \
echo "" && \
echo "========== 音箱 ==========" && \
aplay -l 2>&1 | head -3 && \
echo "" && \
echo "========== 内存 ==========" && \
free -h | grep Mem && \
echo "" && \
echo "========== 机器人服务 ==========" && \
systemctl --user is-active robot.service 2>/dev/null || echo "❌ 未运行" && \
echo "" && \
echo "========== 音乐服务 ==========" && \
systemctl --user is-active netease-api.service 2>/dev/null || echo "❌ 未运行" && \
echo "" && \
echo "========== 磁盘 ==========" && \
df -h / | tail -1
```

**正常输出示例：**
```
========== 文件检查 ==========
7
个 .py 文件

========== 麦克风 ==========
card 3: WebCamera [USB Camera]...
   USB 音频设备已识别

========== 音箱 ==========
card 4: USB2.0 Device [USB Audio]...
   USB 音频设备已识别

========== 内存 ==========
Mem:             3.8G        ...
========== 机器人服务 ==========
active

========== 音乐服务 ==========
active
```

## 🔄 终极手段：重装系统

> 以上方法都试过了还不行？最后的手段。

**SD 卡插回电脑读卡器 → 重新执行第3步（烧录系统）**
**然后重新从第4步开始做一遍。**

> 第二次做会快很多，因为你知道每一步在做什么了。
