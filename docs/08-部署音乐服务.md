# 第八步：部署音乐服务

> 部署网易云音乐的 API 服务器，让机器人能搜歌和放歌。
>
> 📍 **在哪操作：🟢 树莓派窗口**（提示符 `pi@pi-zero:~ $`）

---

## 8.1 安装网易云 API 服务器

> 在树莓派上下载并安装一个"音乐翻译官"，负责把机器人的"搜歌"指令变成网易云音乐能懂的请求。

直接在树莓派窗口输入以下命令（复制整段，粘贴进去）：

```bash
cd ~/robot && git clone https://github.com/Binaryify/NeteaseCloudMusicApi.git netease-server && cd netease-server && npm install
```

**按回车执行。**

> 这条命令做了三件事：
> ① `git clone ...` — 下载音乐服务器代码
> ② `cd netease-server` — 进入下载好的文件夹
> ③ `npm install` — 安装音乐服务器需要的依赖包

**等待 1-2 分钟**，看到类似这样的输出：

```
added 1000+ packages in 30s
```

就安装完了。

**如果安装很慢（国内网络原因）：**

先按 `Ctrl+C` 取消当前命令，然后换国内镜像再试：

```bash
cd ~/robot && rm -rf netease-server && git clone https://github.com/Binaryify/NeteaseCloudMusicApi.git netease-server && cd netease-server && npm config set registry https://registry.npmmirror.com && npm install
```

---

## 8.2 临时启动并测试

> 先手动启动音乐服务器，看看能不能正常工作。

**第一步：启动音乐服务器**

```bash
cd ~/robot/netease-server && node app.js
```

按回车。你会看到一堆日志滚动：

```
Server running @ http://localhost:3000
```

> 注意：这个命令启动后，终端会被"占住"，不能再输入其他命令——这是正常的，说明服务器在运行中。

**第二步：再开一个树莓派窗口来测试**

> 因为刚才的终端被音乐服务器占住了，你需要**新开一个连接**来测试。

**在电脑上的终端里，重新 SSH 连一次树莓派：**

```
① 如果你刚才关了树莓派窗口：
   电脑窗口 → 输入 ssh pi@你的树莓派IP → 密码 raspberry
   
② 如果你还有电脑窗口开着：
   直接在那里输入 ssh pi@你的树莓派IP → 密码 raspberry
```

**第三步：在新树莓派窗口测试服务器**

```bash
curl http://127.0.0.1:3000
```

**✅ 看到一堆 JSON 数据（大括号 {} 开头的内容）→ 音乐服务器跑起来了。**

## 8.3 测试音乐搜索

在新树莓派窗口输入：

```bash
cd ~/robot && python3 -c "
from netease import NetEaseClient
n = NetEaseClient()
songs = n.search('大海')
if songs:
    print(f'搜索 ✅: {songs[0][\"name\"]} - {songs[0].get(\"artist\", \"\")}')
else:
    print('搜索失败，API 服务没启动？')
"
```

**✅ 看到 `搜索 ✅: 大海 - 张雨生` → 音乐搜索正常。**

---

## 8.4 测试音乐播放

确保音箱是通电状态，音量调到适中。

在新树莓派窗口输入：

```bash
cd ~/robot && python3 -c "
from netease import NetEaseClient
from music_player import MusicPlayer
n = NetEaseClient()
m = MusicPlayer(netease_client=n)
print(m.play('大海'))
import time
time.sleep(10)
m.stop()
"
```

**✅ 音箱响起音乐 → 音乐模块完整可用。**

> 这个测试会播放 10 秒后自动停止。

**测试完后**，按 `Ctrl+C` 关掉第一个窗口里的音乐服务器（就是被占住的那个）。

## 8.5 音乐服务开机自启

> 配置好后，树莓派每次开机都会自动启动音乐服务器，不需要手动开。

在 **树莓派窗口** 执行（复制整段命令，粘贴进去）：

```bash
mkdir -p ~/.config/systemd/user/ && cat > ~/.config/systemd/user/netease-api.service << 'EOF'
[Unit]
Description=NetEase Cloud Music API
After=network.target

[Service]
Type=simple
WorkingDirectory=%h/robot/netease-server
ExecStart=/usr/bin/node app.js
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
EOF
```

按回车（这段命令不会显示输出，正常）。

**然后执行：**

```bash
systemctl --user daemon-reload
```

再执行：

```bash
systemctl --user enable --now netease-api.service
```

再执行：

```bash
loginctl enable-linger
```

**验证服务是否启动成功：**

```bash
systemctl --user status netease-api.service
```

**✅ 看到 `active (running)`（绿色字）→ 音乐服务开机自启配置完成！**

> 如果看到红色的 `inactive (dead)` 或 `failed`，检查一下上一步的 rm -rf 是不是把文件夹删了：
> ```bash
> ls ~/robot/netease-server
> ```
> 看有没有文件，如果空了就重新执行 8.1。
